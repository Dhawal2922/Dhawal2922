<summary><b>🧠 AI Architecture Diagram Generator</b> — <i>Your system design co-pilot</i></summary>
<br>

**Problem:** Engineers waste hours sketching architecture diagrams and debating tech stacks before writing a single line of code.

**Solution:** A platform where you describe your product idea (or paste a URL), and it generates complete software architecture diagrams, recommends tech stacks, and produces visual system design blueprints — powered by AI.

**Stack:** React · Python · Django · LLM APIs · Diagram Rendering Engine

**Achievements:**
- End-to-end AI pipeline from natural language → visual architecture
- Smart tech stack recommendations based on project constraints
- Designed as an engineering planning assistant for teams

**Vision:** → AI co-pilot for system design interviews. → SaaS for startups to plan architecture in minutes, not weeks.

**Stack:** React · React Three Fiber · Drei · GSAP · WebGL

<summary><b>☁️ Cloud Storage System</b> — <i>Dropbox-inspired, engineered from scratch</i></summary>
<br>

**Problem:** Building a cloud storage system is one of the most overlooked full-stack challenges — most developers never attempt it end-to-end.

**Solution:** A complete cloud storage backend with user quotas, file compression/decompression, multi-level storage logic, and full file management capabilities.

**Stack:** Python · Django · File I/O · Compression Algorithms · RESTful APIs

**Achievements:**
- User-level quota enforcement and storage accounting
- Compression pipeline reducing storage footprint
- Multi-tier storage hierarchy with access control

**Vision:** → Evolve into a distributed object storage system → Add encryption, versioning, and CDN delivery.

## 💡 Engineering Philosophy

</div>

> **"Every abstraction should earn its complexity. Every feature should justify its existence. Every line should make the product better."**

- **Simplicity scales.** Complex systems fail. The right abstraction beats clever code every time.
- **Performance is a feature.** Every millisecond matters. Users notice even when they can't articulate it.
- **Build for humans first.** Technology is the means. Impact is the measure.
- **Ship, learn, iterate.** A deployed imperfect product beats a perfect unreleased one.
- **Read the source.** The best engineers aren't afraid to go deeper than the docs.
